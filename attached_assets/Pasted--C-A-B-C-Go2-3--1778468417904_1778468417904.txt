# タスク：プロンプトC完了後の動作確認で発見した問題の修正

## 背景

プロンプトA・B・Cが完了し、レッツGo2の3層構造（発注書 → 仕入伝票 → 原価明細）の基本実装は完了している。発注書・仕入伝票・cost_items自動同期・支払査定参照元変更まで動作確認済み。

ただし、本番URL（cost-management-cloud.replit.app）での動作確認中に以下5件の問題を発見した。これらをまとめて修正する。

## 修正対象の問題

### 🔴 問題1（最重要・データ整合性）：仕入伝票ヘッダ合計と明細合計の不一致

**事実**:
- 仕入伝票一覧画面で `ST-20260511-0001` の合計金額が `¥165,000` と表示されている
- 同じ伝票が同期した cost_items は2行で `¥250,000 + ¥90,000 = ¥340,000`
- 明細の単価×数量を計算すると `50×5,000 + 30×3,000 = 340,000`
- つまり**明細側の金額（¥340,000）が正しく、伝票ヘッダの total_amount（¥165,000）がバグっている**

**調査範囲**:
- `artifacts/api-server/src/routes/purchase-invoices.ts` の以下を確認：
  - `POST /api/purchase-invoices`（直接作成時の合計計算）
  - `POST /api/purchase-invoices/from-order/:orderId`（発注書から作成時の合計計算）
  - `PATCH /api/purchase-invoices/:id`（更新時の合計再計算）
- 同様に `artifacts/api-server/src/routes/purchase-orders.ts` の発注書側も、明細合計と一致するか確認（発注書 PO-20260511-0001 も ¥165,000 だが、その元になっている発注明細を見直す）

**修正方針**:
- 仕入伝票ヘッダの `subtotal` / `taxAmount` / `totalAmount` を、明細から再計算してから保存する
- `subtotal = SUM(item.amount)` （税抜小計）
- `taxAmount = SUM(item.amount * item.taxRate / 100)` （消費税）
- `totalAmount = subtotal + taxAmount` （税込合計）
- フロントから受け取った合計値をそのまま信用せず、サーバー側で必ず再計算する
- 同じ修正を発注書（`purchase_orders`）にも適用する
- 既存の発注書 PO-20260511-0001 と仕入伝票 ST-20260511-0001 の合計金額もこの修正後に再計算して正しい値に UPDATE する（マイグレーションスクリプトとして実行）

### 🟡 問題2：「仕入伝票」バッジ・外部リンクアイコンが画面に表示されていない

**事実**:
- プロンプトC完了報告には「仕入伝票由来の行に『仕入伝票』バッジ＋外部リンクアイコンを表示」とある
- 本番URLで工事詳細 → 原価明細タブを見たが、仕入伝票由来の行（sourceType='purchase_invoice'）と手動入力の行（sourceType='manual'）の見た目に**違いがない**

**調査範囲**:
- 工事詳細の原価明細タブを実装しているコンポーネント（`artifacts/genka-kanri/src/` 配下の `detail.tsx` 相当のファイル）
- 各行のレンダリングロジックで `sourceType === 'purchase_invoice'` の判定を確認
- API（`GET /api/cost-items`）のレスポンスに `sourceType` が含まれているか実機で確認（`curl` か Network タブで）

**修正方針**:
- API レスポンスに `sourceType` が含まれていない場合は、`cost-items.ts` の SELECT クエリを修正
- フロント側で判定ロジックが正しく動いていない場合は、JSX を修正
- バッジは**明確に視認できる色とサイズ**にする（例: 青色のバッジ、`仕入伝票` の文字 + 外部リンクアイコン）

### 🟡 問題3：仕入伝票由来の cost_items 行で削除ボタンが有効になっている

**事実**:
- プロンプトC完了報告には「仕入伝票由来の行の削除ボタンを無効化」とある
- 本番URLで確認したところ、`sourceType='purchase_invoice'` の行でも削除ボタン（ゴミ箱アイコン）が普通にクリックできる状態

**修正方針**:
- 原価明細タブのコンポーネントで、各行の削除ボタンに `disabled={row.sourceType === 'purchase_invoice'}` を追加
- 無効化されたボタンは見た目もグレーアウト
- ツールチップ等で「仕入伝票から自動同期された原価です。削除するには仕入伝票側を編集してください」と表示できればなお良い（任意）

### 🟡 問題4：仕入伝票一覧から個別伝票の編集画面に遷移できない

**事実**:
- 仕入伝票一覧画面で、伝票番号（`ST-20260511-0001` のリンク表示）をクリックしても画面遷移しない
- `/purchases?id=1` のURLでアクセスすると、編集モードではなく**空の新規作成フォーム**が表示される

**修正方針**:
- 既存 `/purchases` ページに編集モード対応を追加：
  - URL クエリパラメータ `?id={invoiceId}` を読み取って、`GET /api/purchase-invoices/:id` でデータ取得
  - フォームに既存値を埋めて編集できるようにする
  - 保存時は `PATCH /api/purchase-invoices/:id` を呼ぶ（新規作成時は POST）
- 仕入伝票一覧の伝票番号リンクの遷移先を `/purchases?id={invoiceId}` に設定

### 🟡 問題5：ダッシュボードKPIに仕入伝票分が反映されていない

**事実**:
- ダッシュボード画面の「実績原価合計」が `¥123,316,000` のままで、町田邸の新規追加分（cost_items 経由で同期された ¥340,000、または問題1修正後の正しい金額）が反映されていない可能性がある
- ただし `cost_items` には同期データが入っているので、ダッシュボードの集計クエリが何を見ているか確認が必要

**調査範囲**:
- ダッシュボード関連エンドポイント:
  - `/api/dashboard/cost-by-category`
  - `/api/dashboard/monthly-costs`
  - 「実績原価合計」を返している箇所
- これらが `cost_items` を全件 SELECT して合計しているか、あるいは古いキャッシュやSubqueryを使っているか確認

**修正方針**:
- ダッシュボード集計クエリが `cost_items` の全件（`sourceType` 問わず）を SUM していることを確認
- そうなっていなければ、`sourceType` 区別なしの集計に修正
- 「実績原価合計」は `cost_items.amount` の SUM（全工事横断）

## 制約

- DBスキーマは変更しない（プロンプト1で確定済み）
- 既存の22件の手動入力 cost_items（`source_type='manual'`）は変更しない
- フェーズの順序として、**問題1（金額不整合）を最優先**で修正する（これがバグっていると以降の確認も信用できない）

## 実装手順

### ステップ1：問題1（金額不整合）を修正

`artifacts/api-server/src/routes/purchase-invoices.ts` と `purchase-orders.ts` を修正：
1. 明細合計から `subtotal`、`taxAmount`、`totalAmount` を再計算する関数を作る
2. POST・PATCH・from-order の各エンドポイントで、保存前にこの関数を呼んでヘッダ値を上書きする
3. 既存の `PO-20260511-0001` と `ST-20260511-0001` のヘッダ金額を、明細から再計算して UPDATE する一回限りのスクリプトを実行（または手動でSQL UPDATE）

### ステップ2：問題2・3を修正（フロント）

原価明細タブのコンポーネントを修正：
1. API レスポンスに `sourceType` が含まれることを確認（含まれていなければ API も修正）
2. `sourceType === 'purchase_invoice'` の行にバッジ + 外部リンクアイコンを表示
3. 同行の削除ボタンを `disabled` にする

### ステップ3：問題4を修正（仕入伝票編集モード）

既存 `/purchases` ページを編集モード対応に：
1. URL クエリパラメータ `?id={invoiceId}` の読み取り
2. データ取得・フォーム初期化
3. 保存時の API 切り替え（POST or PATCH）
4. 仕入伝票一覧の伝票番号リンクを `/purchases?id={invoiceId}` に設定

### ステップ4：問題5を確認・修正（ダッシュボード）

1. 現状のダッシュボード集計クエリを確認
2. `cost_items` 全件 SUM になっていることを確認
3. なっていなければ修正

```bash
pnpm run typecheck
```

エラーゼロを確認後、以下を実機確認：
1. 仕入伝票一覧の `ST-20260511-0001` の合計金額が `¥340,000` に修正されている（または明細合計と一致する正しい値）
2. 工事詳細 → 原価明細タブで、仕入伝票由来の行に青いバッジが見える
3. 仕入伝票由来の行の削除ボタンが無効化されている
4. 仕入伝票一覧から伝票番号をクリックすると、編集モードで `/purchases` が開く
