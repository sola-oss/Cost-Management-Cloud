# プロンプト①：実行予算明細の行編集ボタンクラッシュ修正

## 背景

プロジェクト詳細ページ（`/projects/:id`）の「実行予算」タブにある実行予算明細テーブルで、各行の右端にある編集アイコン（鉛筆マーク）をクリックすると、**画面が真っ白になりReactがクラッシュする**バグが発生しています。

### 発生環境
- 本番URL: https://cost-management-cloud.replit.app/
- 該当画面: プロジェクト詳細ページの「実行予算」タブ
- 該当UI: 実行予算明細テーブル各行の鉛筆アイコンボタン

### コンソールエラー
```
TypeError: a.map is not a function
    at https://cost-management-cloud.replit.app/assets/index-BUHHXIjy.js:121:161194
    at Array.map (<anonymous>)
    at s6e (...)
    at d2 (...)
    at P2 (...)
```

### 再現手順
1. 工事一覧から任意のプロジェクト（例: 町田邸 新築工事）を開く
2. 「実行予算」タブをクリック
3. 実行予算明細テーブルの任意の行の右端にある **鉛筆アイコン**（編集ボタン）をクリック
4. **画面が真っ白になる**

### 推定原因
今回の Task #48「実行予算 → 発注書 一括作成」で `budget_items` テーブルに `vendor_id` / `purchase_order_id` / `purchase_order_item_id` の3カラムを追加した影響で、行編集モーダル/ダイアログのReactコンポーネントが新フィールドを配列前提で読みに行こうとして `.map()` を呼んでいるが、既存レコードがNULLのため落ちている可能性が高い。

## 依頼内容

### ステップ1：事前調査（コード変更前に必ず実施）

以下を調査して報告してください：

1. **実行予算明細テーブルの鉛筆アイコン（編集ボタン）のクリックハンドラ** が何を呼んでいるか特定
   - 対象ファイル候補: `artifacts/genka-kanri/` 配下の実行予算タブ関連コンポーネント
   - 編集モーダル/ダイアログのコンポーネント名と場所
2. その編集コンポーネント内で `.map()` を呼んでいる箇所を全て列挙
3. 各 `.map()` が読んでいるpropsまたはstateが、新規追加カラム（`vendor_id` / `purchase_order_id` / `purchase_order_item_id`）に関係するか確認
4. **配列型を期待しているのに NULL/undefined が渡される可能性のある箇所** を特定

調査結果を**コードスニペットと共に**報告してください。実装はその後で。

- 修正前後のコード差分（diff形式）
- なぜその修正で問題が解決するかの説明

### ステップ3：実装

## 制約条件

- **既存の3層構造（発注書 → 仕入伝票 → 原価明細）の動作を壊さないこと**
- **既存の一括発注機能（Task #48）の動作を壊さないこと**
- **typecheckエラーを発生させないこと**（`pnpm typecheck` がクリーンに通る）
- **修正は最小限**にする（リファクタリングや関係ない箇所の整理は今回は不要）
- **架空のコードや推測で書かない**こと。実際のコードを `view` で確認してから書く

## 完了条件

1. 鉛筆アイコンをクリックしても画面がクラッシュしない
2. 編集モーダル/ダイアログが正常に開く
3. 編集モーダルから既存の項目（工種、仕入先、金額等）を編集・保存できる
4. typecheck がクリーンに通る
5. コンソールに新たなエラーが出ない

## 補足

このバグは Task #48 のリグレッションの可能性が高いです。**Task #48 で `budget_items` テーブルに追加した3カラム（`vendor_id` / `purchase_order_id` / `purchase_order_item_id`）の扱いを、編集モーダル側でフォローし忘れている**ことが推定原因なので、その観点で重点的に調査してください。

なお、近日中に別プロンプトで「実行予算入力画面の再設計」を予定しており、その過程で鉛筆アイコン経由の行編集モーダルは廃止する予定です。**今回の修正はあくまで暫定対応**として、最小限の修正でクラッシュを止めることが目的です。大規模なリファクタリングは不要です。
