# プロンプト⑤-2：支払管理画面（/payments）のリスト改善＋振込対象選択UI

## 背景

`/payments` 画面は現状リストビューで支払状況を管理できますが、以下のUX改善余地があります。さらに、後続タスク（Zengin形式の振込データ出力）で使う「振込対象の選択UI」を、今回のリスト改善と一緒に作ることで、UIを二度作らずに済ませます。

## 現状の `/payments` 画面（把握済み）

- タイトル：「支払管理」
- サマリーカード：支払総額・未払・支払済・支払率
- ステータスタブ：すべて / 未払 / 一部払 / 支払済 / 期日超過
- 工事フィルタ：全工事 ▼
- リスト：状態 / 工事 / 支払先 / 内容 / 支払金額 / 支払期日 / 支払日
- データソース：`payment_assessments`（支払査定）と `purchase_invoices`（仕入伝票）の混在表示

   - 「状態」（未払/一部払/支払済/期日超過）の判定ロジック
   - 「支払期日」「支払金額」「支払日」「未払額」を取得する経路
3. **既存ソート・フィルタの実装**
   - 現在のデフォルトソート順
   - フィルタタブの実装方式

調査結果を**コードスニペットと現状サマリー**で報告してください。実装はその後で。

### ステップ2：修正方針の提示

調査結果を踏まえて、以下を提示してください：

1. デフォルトソート変更の影響範囲
2. 期日超過行のスタイル適用方法
3. 警告バナーコンポーネントの配置
4. チェックボックス列とクイック選択ボタンの追加方法
5. 各変更の差分（diff形式）

### ステップ3：実装

以下4つの改善を実装してください。

#### 改善A：デフォルトソートを支払期日順に
- リストのデフォルトソートを **支払期日の昇順（早い順）** に変更
- 期日未設定（NULL）の行は末尾に
- 既存のソート機能（列ヘッダクリックなど）があれば、それも引き続き動作すること

#### 改善B：期日超過行の視認性強化
- **期日超過 かつ 未払/一部払** の行の背景色を薄い赤（例: `bg-red-50` ）にする
- 行ホバー時はやや濃い赤（例: `bg-red-100` ）
- すでに「⚠️」マークが期日列に出ているのでそれは維持
- **画面上部に警告バナー** を表示：
  - 期日超過件数が1件以上のときに表示
  - 例：「⚠️ 期日超過の支払が **3件** あります（合計 ¥709,600）。早急にご確認ください。」
  - バナー色は警告色（赤系の薄背景＋濃赤テキスト）
  - 期日超過が0件のときはバナー非表示

#### 改善C：チェックボックス列と振込対象選択UI（後続Zengin出力用）

##### C-1. チェックボックス列を追加
- リスト先頭にチェックボックス列を追加
- **チェックボックスの活性条件**：
  - 状態が「未払」かつ 仕入先が設定されている行 → 活性
  - 上記以外（一部払・支払済・期日超過の支払済、仕入先未設定）→ 非活性（グレーアウト）
  - 非活性の場合、ホバー時にツールチップで理由表示（例：「支払済の行は対象外です」「仕入先が未設定です」）
- ヘッダー行の全選択チェックボックスは、「現在表示中で活性な行」のみを対象

##### C-2. クイック選択ボタンを追加（4種類）
リスト上部、サマリーカードと工事フィルタの間に配置：

- **「今週支払い」**：今週中（今日〜今週日曜まで）に期日が来る未払行を全選択
- **「今月支払い」**：今月末までに期日が来る未払行を全選択
- **「期日超過のみ」**：期日が今日より前の未払行を全選択
- **「全未払」**：状態が「未払」の全行を選択

ボタンクリックで該当行のチェックボックスが切り替わる。再クリックで解除はしない（あくまでプリセット適用）。

##### C-3. 選択サマリーと「振込データを出力」ボタン（仮配置）
- リスト上部の右側に「**選択: N件 / ¥XXX,XXX**」を表示
- その横に **「📤 振込データを出力」ボタン** を配置
- このボタンは**今回はクリックしてもまだ動作しない**（プロンプト⑤-3で実装）。代わりに **「準備中です」のトースト表示** にしておく
- 0件選択時はボタン非活性

#### 改善D：（タスク説明のため改善Dとして追加）支払期日が今日のものは黄色背景
- 期日超過ではないが**期日が今日**の未払行を **薄い黄色（例: `bg-yellow-50`）** で強調表示
- 期日超過の赤と区別がつくこと（期日超過は赤、当日は黄色）
- 「今日が期日です」のような明示は不要（背景色で十分）

## 制約条件

- **既存の3層構造・一括発注・仕入先選択ドロップダウンの動作を壊さないこと**
- **`/payments` 画面で他に表示されている全てのデータ（状態タブ・サマリーカード・フィルタ）の動作を壊さないこと**
- **typecheckエラーを発生させないこと**（`pnpm typecheck` がクリーンに通る）
- **既存のフロントエンドデザイントークン（CSS変数・Tailwindユーティリティ）を使用すること**
- **架空のコードや推測で書かない**こと

## 完了条件

1. `/payments` を開くと、リストが**支払期日の早い順**で並んでいる
2. 期日超過の未払行が**薄い赤背景**で表示されている
3. 期日が今日の未払行が**薄い黄色背景**で表示されている
4. 期日超過件数が1件以上のとき、画面上部に**警告バナー**が表示される
5. リスト先頭に**チェックボックス列**があり、未払行のみチェック可能
6. リスト上部に**クイック選択ボタン4種**（今週支払い / 今月支払い / 期日超過のみ / 全未払）が配置されている
7. 右側に「**選択: N件 / ¥XXX,XXX**」と「📤 振込データを出力」ボタンが表示される
8. 「振込データを出力」ボタンをクリックすると「準備中です」のトーストが出る
9. typecheck がクリーンに通る

## 補足

- 「振込データを出力」ボタンの実体（Zengin形式CSV生成）は別タスクで実装します。今回はUI配置のみ。
- 期日判定の「今日」「今週」「今月」はサーバ時刻ではなくユーザーのローカル時刻でOKです。
- 既存の状態タブ（すべて/未払/一部払/支払済/期日超過）は変更不要です。
