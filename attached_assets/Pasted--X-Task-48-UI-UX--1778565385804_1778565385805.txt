# プロンプト②：仕入先入力のマスタ選択型化（パターンX）

## 背景

Task #48（実行予算 → 発注書 一括作成機能）の通しテスト中に、仕入先設定のUI/UXに以下の問題が判明しました。

### 現状の問題

1. **2つの編集画面で仕入先入力のUIが不統一**
   - **実行予算タブ（`/projects/:id` の「実行予算」タブ）の鉛筆アイコン経由のインライン編集**：
     → 仕入先マスタからのドロップダウン選択（既に実装済み・正常動作）
   - **「実行予算入力画面」（`/projects/:id/budgets`）のグリッド編集**：
     → 仕入先コード／仕入先名が**自由テキスト入力**（マスタ選択ではない）

2. **マスタ選択ドロップダウンに不足機能がある**
   - 「＋ 新しい仕入先を登録」リンクがない（マスタにない仕入先を選びたい時、別画面に行く必要がある）
   - 検索機能がない（仕入先が多くなると探しづらい）

3. **既存データのマイグレーション課題**
   - `budget_items` テーブルの既存レコードには、自由テキストの `supplierName` が入っているが `vendor_id` は NULL のまま
   - これらが「仕入先設定済み」と判定されないため、一括発注機能のチェックボックスが有効化されない

### 目指す姿（パターンX：マスタ駆動）

- 仕入先の入力は **すべてマスタ選択型ドロップダウンに統一**
- マスタにない仕入先は、ドロップダウン内の「＋ 新しい仕入先を登録」リンクから即マスタに登録 → そのまま選択状態に
- 既存データはマイグレーションで自動マッチング、未マッチは手動修正を促す
