# 機能追加：発注書PDF出力

発注書（purchase_orders）のPDF出力機能を追加してほしい。
既存の請求書PDF実装（invoices）と同じスタック（jspdf + jspdf-autotable）で揃えること。

## 事前調査（必須）

実装に着手する前に、必ず以下を読んで方針を把握してから進めること：

1. `artifacts/genka-kanri/` 配下の請求書PDF出力の実装箇所を grep で探す
   - キーワード：`jspdf`, `autoTable`, `request-invoice` または `invoice-pdf` 系のファイル
2. 請求書PDFで使われている日本語フォント（NotoSansJP 等）の組み込み方を確認
3. `lib/db/src/schema/` で以下のスキーマを確認
   - purchase_orders（発注書）
   - purchase_order_items（発注書明細）
   - vendors（仕入先）
   - projects（工事）
   - company_settings（会社設定）

請求書PDFのコードと同じ関数命名規則・ファイル構成・フォント読み込み手法をそのまま踏襲すること。
新規ライブラリの追加は不要。

## 実装範囲

### 1. PDF生成関数の追加

ファイル：請求書PDFと同じディレクトリに `purchase-order-pdf.ts`（命名は既存に合わせて調整可）を作成。

引数：
- 発注書ID（id: string）

処理：
1. 発注書本体・明細・仕入先・工事・会社設定を取得
2. A4縦・日本語フォント対応のPDFを生成
3. `blob` を返す（保存・印刷は呼び出し側で実施）

### 3. UI 統合

発注書の以下2箇所に「PDF出力」ボタンを追加：

- **発注書一覧画面の各行のアクション列**（編集・削除ボタンの隣に「PDF」アイコンボタン）
- **発注書編集モーダルのフッター**（「保存」ボタンの隣に「PDF出力」ボタン）

クリック時の挙動：
- PDFを生成し、新規タブで開く（または `window.open(URL.createObjectURL(blob))` で表示）
- ファイル名: `発注書_PO-YYYYMMDD-NNNN_仕入先名.pdf`

### 4. 細かい仕様

- 数値は3桁カンマ区切り（¥1,000,000 形式）
- 日付は「2026年5月11日」形式
- 消費税は明細の `tax_rate`（あれば）で計算、なければ一律10%
- 軽減税率8%が混在する場合は税率ごとに分けて表示（既存の請求書PDFと同じロジックを流用）
- 仕入先住所が登録されていれば、宛先の下に小さく表示
- 自社情報は company_settings テーブルから取得（未設定時はその行をスキップして体裁を崩さない）

## 制約

- 既存の請求書PDFの体裁・フォント・余白の取り方を踏襲する
- 新規パッケージのインストールは不要（jspdf, jspdf-autotable は既に入っている）
- typecheck と build が通ること（`pnpm run typecheck` でエラーが出ないこと）
- 既存機能（請求書PDF、発注書CRUD、仕入伝票連動）に影響を与えないこと

1. 発注書一覧から「PDF」ボタンを押すと、A4縦の日本語PDFが新規タブで開く
2. 発注書編集モーダルからも同じPDFが開ける
3. 仕入先名・工事名・明細・税額・自社情報が正しく反映されている
4. typecheck が通る
